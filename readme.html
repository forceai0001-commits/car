1. 系統簡介與用途

本系統為一套「輕量級、快速落地」的倉儲車輛預約與現場管理工具。採用 WebApp 架構（網頁版），無需下載 App 即可使用。

核心目的：

  • 分流管理：降低現場車輛同時湧入造成的碼頭擁塞。
  • 數位紀錄：建立可追溯的車輛到場紀錄（包含照片與時間）。
  • 即時調度：提供管理者即時掌握每日車輛負載狀況。

3. 司機端操作說明

操作流程圖

必填欄位指南

  • • 作業類型：請依實際狀況選擇。
  • • ETA：系統預設為 +1 小時，可手動調整。
  • • 聯絡電話：手機或 Line ID 至少填一項。
  • • 現場照片：必須上傳 1 張，建議拍攝清晰車牌。

常見問題 (Driver)

Q: 照片上傳很久？

A: 因系統需將照片轉存至雲端硬碟，請耐心等待約 3-5 秒，直到畫面顯示「登記成功」。

Q: 提交後需要修改嗎？

A: v0 版本暫不開放司機自行修改，若填錯請直接通知現場管理人員協助。

5. 系統邏輯：容量分流

本系統採用「雙軌容量 (Dual Bucket)」設計。STD 為主要預約池，PROJ 為例外管理/現場池，確保調度彈性。

6. v0 版本限制

  • 無帳號體系：

    目前不記錄操作人員 ID，所有變更視為「管理員」操作。

  • 無自動辨識 (OCR)：

    車牌 OCR 功能尚未啟用，目前依賴司機手動輸入與人工核對照片。

  • 資料安全：

    目前介面不提供刪除功能（防止誤刪），若需刪除錯誤單據，請聯絡 IT 人員。

7. 部署教學：將 GAS 部署為 Web App

本系統採用「Google Sheet 作為資料庫、GAS 作為 API」的架構。以下步驟可完成後端部署，讓前端網頁能透過 Web App URL 呼叫 API（create/list/summary...）。

1

準備 Google Sheet（資料庫）

  • 建立或開啟專案用 Google Sheet（建議為獨立一份，不與其他用途混用）。
  • 進入「檔案 → 試算表設定」，確認時區為：台北標準時間（GMT+8 / Asia/Taipei）。
  • ※ 這個設定非常重要，會影響管理端「依日期查詢」是否正常。
2

建立 Apps Script 專案

  • 在 Google Sheet 內，點選：擴充功能 → Apps Script。
  • 建立一個新專案後，找到 Code.gs（或主程式檔）。
  • 將後端程式碼貼上並儲存。
  • 確認程式碼中是否使用 SpreadsheetApp.getActiveSpreadsheet() 以綁定目前試算表。
3

第一次授權

  • 在 Apps Script 編輯器上方，點選「執行」任一個函式（或直接部署也會觸發）。
  • 系統會彈出權限要求，請選擇你的 Google 帳號並點選「進階」→「前往...」以同意權限。
  • ※ 權限包含：讀寫試算表、建立/讀取 Drive 檔案（用於照片上傳）。

部署為 Web App（取得 API URL）

右上角點選：部署 → 新增部署 (New deployment)，類型選擇「網頁應用程式 (Web app)」。

執行身分： 我 (Me) - 代表使用你的權限寫入 DB
可存取對象： 任何人 (Anyone) - v0 先求上線，方便前端呼叫

點選「部署」後，複製 Web App URL（即為 API_BASE_URL）。

前端設定與驗證

  • 前端設定：將 HTML 中的 API_BASE_URL 變數替換為上一步取得的網址。
  • 快速驗證：
    1. 用司機端提交一筆資料（含照片）。
    2. 確認 Google Sheet 的 Bookings 分頁有新增資料。
    3. 確認 Google Drive 有新增照片檔案。
